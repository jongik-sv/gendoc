## 5. 컨텐츠 유형별 가이드

### 5.1 TEXT (텍스트 중심)

**해당 슬라이드**: 9-19

**구성 요소**:
- 제목 (Action Title): 20pt, 최대 2줄
- 본문: 12pt, 들여쓰기 3단계 지원
- 강조 박스 (선택): 직사각형 AUTO_SHAPE

**레이아웃 패턴**:
```
[1열 전체 너비]     - 긴 설명, 정책 안내
[2열 분할]          - 비교, 전/후
[3열 분할]          - 단계별 설명
```

**적합한 내용**: 정책 설명, 프로세스 안내, 긴 텍스트 콘텐츠

---

### 5.2 BOXES (박스형)

**해당 슬라이드**: 21-26

**구성 요소**:
- AUTO_SHAPE (RECTANGLE): 정보 박스
- FREEFORM: 화살표, 연결선
- TEXT_BOX: 라벨, 설명

**레이아웃 패턴**:
```
[2×2 그리드]        - 4가지 항목 비교
[3×1 가로]          - 3단계 프로세스
[1×3 세로]          - 우선순위 목록
[자유 배치]         - 관계도
```

**박스 스타일**:
| 유형 | 배경색 | 테두리 | 용도 |
|------|--------|--------|------|
| 기본 | 흰색 | 회색 1px | 일반 정보 |
| 강조 | 연청색 (#B7D0D4) | 없음 | 핵심 포인트 |
| 경고 | 연빨강 | 빨강 1px | 주의사항 |

**적합한 내용**: 특징 비교, 구성 요소 설명, 카테고리 분류

---

### 5.4 DIAGRAM (다이어그램)

**해당 슬라이드**: 34-81

**주요 도형**:
| 도형 | 코드 | 용도 |
|------|------|------|
| 직사각형 | RECTANGLE | 프로세스 박스, 단계 |
| 원 | OVAL | 시작/끝, 핵심 요소 |
| 연결선 | FLOWCHART_CONNECTOR | 흐름 연결 |
| 화살표 | LINE/FREEFORM | 방향 표시 |

**다이어그램 유형**:
```

[벤 다이어그램]      - 관계, 교집합
[피라미드]          - 우선순위, 계층
[순환도]            - 반복 프로세스
[타임라인]          - 일정, 마일스톤
[매트릭스]          - 2×2 분석
```

**적합한 내용**: 프로세스 설명, 조직 구조, 관계 표현, 전략 프레임워크

## 8. LLM 사용 시 주의사항

### 8.1 텍스트 입력 규칙

1. **Action Title**: 최대 2줄, 줄당 약 40자 (한글 기준)
2. **본문**: 들여쓰기 3단계까지 지원
3. **테이블 셀**: 한 셀당 50자 이내 권장
4. **차트 라벨**: 간결하게 (10자 이내)

### 8.2 금지 사항

- ❌ 폰트 변경 (본고딕 유지)
- ❌ 색상 팔레트 외 색상 사용
- ❌ 정의된 크기 외 폰트 크기 사용
- ❌ 컨텐츠 영역 밖 요소 배치
- ❌ 로고/CI 수정

### 8.3 권장 사항

- ✅ 일관된 들여쓰기 수준 사용
- ✅ 간지로 섹션 명확히 구분
- ✅ 한 슬라이드 = 하나의 핵심 메시지
- ✅ 차트는 데이터 3-5개 계열 이내
- ✅ 박스는 페이지당 4-6개 이내
