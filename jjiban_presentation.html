Local & File-based

데이터베이스 없는 순수 파일 기반 아키텍처

No Database

별도의 DB 설치 없이 JSON/Markdown 파일로 모든 데이터를 저장합니다.

.jjiban/ projects/
├── task-001.json
└── wbs.md

Git Sync

git push/pull이 곧 백업이자 팀 동기화 방식입니다.

  • 오프라인 작업 가능
  • 완벽한 히스토리 관리
  • 벤더 종속성 탈피

LLM Integration

AI가 프로젝트의
동료가 됩니다

Claude Code, Gemini CLI 등과 자연스럽게 연동되어, LLM이 직접 파일 시스템을 통해 Task 상태를 변경하고 문서를 작성합니다.

Key Features

WBS 트리 뷰

Project > WP > Activity > Task로 이어지는 계층형 작업 관리 시스템

칸반 보드

직관적인 상태 시각화 및 드래그 앤 드롭을 통한 손쉬운 관리

워크플로우 엔진

사용자 정의 가능한 상태 전이 규칙 및 자동화 트리거

문서 관리

Markdown 기반의 설계 및 구현 문서를 코드와 함께 통합 관리

Distributed JSON

개별 Task의 상세 정보 및 상태를 분산 저장하여 충돌 방지

Workflow Engine

workflows.json 
{
  "name": "Development",
  "states": ["To Do", "In Progress", "Done"],
  "transitions": [
    {
      "from": "To Do",
      "to": "In Progress",
      "action": "create_branch"
    },
    {
      "from": "In Progress",
      "to": "Done",
      "action": "create_pr"
    }
  ]
}

유연성 (Flexibility)

JSON 설정을 통해 상태 및 전이 규칙을 자유롭게 정의

자동화 (Automation)

상태 변경 시 문서 템플릿 생성, 브랜치 생성 등 자동 수행

확장성 (Scalability)

개발, 결함, 인프라 등 카테고리별로 다른 워크플로우 적용

Roadmap
